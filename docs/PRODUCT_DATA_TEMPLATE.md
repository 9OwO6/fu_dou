# Happy Beans 商品资料模板

> 用途：店主为每个商品复制并填写一份。  
> 第一版规则：中文必填，英文留空；价格仅 CAD；每个具体规格组合独立管理。  
> 提醒：下文三个“结构验证样例”的名称、SKU、价格和库存均为测试数据，不可直接用于上线。

## 1. 店主填写模板

### A. 基本资料

| 字段 | 必填 | 填写内容 |
|---|---:|---|
| 店主内部商品编号 | 是 | 例：`HB-P0001`；仅用于资料整理 |
| 中文商品名 | 是 |  |
| 英文商品名 | 否 | 第一版可留空 |
| 建议 slug | 否 | 小写英文和连字符；最终由系统校验 |
| 商品状态 | 是 | 草稿 / 上架 / 归档 |
| 中文短描述 | 是 | 建议 30–60 字，用于商品卡和摘要 |
| 中文完整描述 | 是 | 外观、用途、使用场景；不要写无法证明的承诺 |
| 分类 | 是 | 可多选；分类名称需来自已确认清单 |
| 材质 | 是 | 按包装或供应商可靠资料填写；不确定则写“待确认” |
| 商品整体尺寸 | 视情况 | 写单位，例如 `约 350 mL`、`40 × 60 cm` |
| 包装内容 | 是 | 顾客实际收到的物品和数量 |
| 使用/护理说明 | 视情况 | 例如是否可用于洗碗机、清洁方法；必须有依据 |
| 安全/注意事项 | 视情况 | 易碎、年龄、温度等真实注意事项 |
| 推荐/新品 | 否 | 是否推荐；计划发布时间 |
| SEO 标题/描述 | 否 | 可在后台阶段补充 |

### B. 规格定义

没有顾客可选规格时，在“规格名称”写 `无`，并仍在 C 表填写一个 `默认` variant。

| 规格顺序 | 规格名称（中文） | 内部 key 建议 | 规格值（按展示顺序） | 是否需要色块 | 对应图片说明 |
|---:|---|---|---|---:|---|
| 1 |  |  |  |  |  |
| 2 |  |  |  |  |  |
| 3 |  |  |  |  |  |

填写规则：

- 规格只描述顾客作出购买选择所需的维度，例如款式、颜色、尺寸。
- 不把材质、容量等所有属性都误设为规格；如果顾客不能选择，就放在基本资料。
- 规格值名称要稳定，例如不要同时出现 `蓝` 和 `蓝色`。
- 不销售的组合可以不建立，或建立后标为停用；所有在售组合必须唯一。

### C. 可购买规格组合（variants）

| 组合序号 | 规格组合 | SKU（唯一） | 售价 CAD | 原价 CAD（可空） | 特价开始/结束（可空） | 库存 | 启用 | 关联图片 |
|---:|---|---|---:|---:|---|---:|---:|---|
| 1 |  |  |  |  |  |  | 是/否 |  |

填写规则：

- 金额填写数字并保留两位小数，不写 `$`；系统展示时格式化为 CAD。
- 原价必须高于售价；没有特价则原价和活动时间均留空。
- 库存为大于等于 0 的整数。
- 即使多个组合价格相同，也要逐行填写，避免遗漏。
- SKU 在全店唯一，不包含空格；不要复用已归档商品的 SKU。
- 没有顾客可选项的商品仍有一行 `默认` variant，以承载 SKU、价格和库存。

## 2. 结构验证样例一：猫狗茶杯（款式 + 颜色）

> 仅验证模板能表达 2×2 的四个组合，不代表真实商品事实。

### A. 基本资料摘要

| 字段 | 测试值 |
|---|---|
| 店主内部商品编号 | `DEMO-MUG-01` |
| 中文商品名 | 猫狗茶杯（结构测试） |
| 状态 | 草稿 |
| 短描述 | 用于验证款式和颜色双规格的测试商品。 |
| 分类 | 杯具（待店主确认正式分类） |
| 材质/容量/护理 | 待真实包装或供应商资料确认 |

### B. 规格定义

| 顺序 | 规格名称 | 内部 key | 规格值 | 色块 | 图片策略 |
|---:|---|---|---|---:|---|
| 1 | 款式 | `style` | 猫猫、狗狗 | 否 | 每个款式至少一张图 |
| 2 | 颜色 | `color` | 浅黄、浅蓝 | 是 | 外观差异明显时关联到具体组合 |

### C. 四个组合

| 组合 | SKU | 售价 CAD | 原价 CAD | 库存 | 启用 | 验证重点 |
|---|---|---:|---:|---:|---:|---|
| 猫猫 / 浅黄 | `DEMO-MUG-CAT-YL` | 18.00 | 22.00 | 3 | 是 | 有特价且有库存 |
| 猫猫 / 浅蓝 | `DEMO-MUG-CAT-BL` | 22.00 |  | 0 | 是 | 同商品不同价格且售罄 |
| 狗狗 / 浅黄 | `DEMO-MUG-DOG-YL` | 20.00 |  | 5 | 是 | 独立库存 |
| 狗狗 / 浅蓝 | `DEMO-MUG-DOG-BL` | 20.00 |  | 2 | 否 | 有库存但组合停用 |

验证结论：模板可表达任意两维规格组合、组合级图片、不同价格、特价、售罄和停用状态。真实录入时必须替换全部 `DEMO` 数据。

## 3. 结构验证样例二：尺寸地毯（单一尺寸规格）

> 仅验证同一商品不同尺寸的独立价格和库存。

### A. 基本资料摘要

| 字段 | 测试值 |
|---|---|
| 店主内部商品编号 | `DEMO-RUG-01` |
| 中文商品名 | 尺寸地毯（结构测试） |
| 状态 | 草稿 |
| 分类 | 地毯（待店主确认正式分类） |
| 材质/护理 | 待真实标签确认 |

### B. 规格与组合

规格名称：`尺寸`；内部 key：`size`。

| 规格组合 | SKU | 售价 CAD | 原价 CAD | 库存 | 启用 |
|---|---|---:|---:|---:|---:|
| 40 × 60 cm | `DEMO-RUG-4060` | 24.00 |  | 4 | 是 |
| 50 × 80 cm | `DEMO-RUG-5080` | 34.00 | 40.00 | 2 | 是 |
| 60 × 90 cm | `DEMO-RUG-6090` | 46.00 |  | 0 | 是 |

验证结论：模板可表达单一规格维度、带单位的规格值、尺寸级价格/特价和库存。

## 4. 结构验证样例三：无顾客可选项的单规格商品

> 示例商品类型使用“家居摆件”只是为了验证结构，店主需要换成真实商品。

### A. 基本资料摘要

| 字段 | 测试值 |
|---|---|
| 店主内部商品编号 | `DEMO-DECOR-01` |
| 中文商品名 | 单规格家居摆件（结构测试） |
| 状态 | 草稿 |
| 分类 | 家居摆件（待店主确认正式分类） |
| 包装内容 | 1 件（测试值） |

### B. 规格与默认 variant

顾客可选规格：`无`。

| 内部组合 | SKU | 售价 CAD | 原价 CAD | 库存 | 启用 | 前台行为 |
|---|---|---:|---:|---:|---:|---|
| 默认 | `DEMO-DECOR-DEFAULT` | 12.00 |  | 8 | 是 | 直接显示价格和库存，不显示规格选择器 |

验证结论：模板不会强迫店主为单规格商品创建虚假的颜色或尺寸，同时仍能保持 SKU、价格和库存都归属于 variant。

## 5. 三类样例覆盖矩阵

| 能力 | 猫狗茶杯 | 尺寸地毯 | 单规格商品 |
|---|---:|---:|---:|
| 两个规格维度 | 是 | 否 | 否 |
| 一个规格维度 | 否 | 是 | 否 |
| 无顾客可选规格 | 否 | 否 | 是 |
| 组合级 SKU/价格/库存 | 是 | 是 | 是 |
| 组合级特价 | 是 | 是 | 可支持 |
| 售罄状态 | 是 | 是 | 可支持 |
| 停用无效组合 | 是 | 可支持 | 可支持 |
| 规格关联图片 | 是 | 可支持 | 使用商品通用图 |

结论：该模板足以作为 Phase 3 数据模型 seed 设计和 Phase 5 后台录入验收的输入，不需要为这三种商品建立三套平行模型。
