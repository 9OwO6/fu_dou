## 1. 已冻结的 MVP 边界

以下内容可直接作为 Phase 1 的工程输入和 Phase 2 的视觉范围输入：

| 主题 | 冻结决定 |
|---|---|

| 首发语言 | 仅中文；路由、内容和数据结构预留英文 |

## 2. 中文文案语气基线

- 语气：温暖、清楚、亲切，像本地小店店主在帮助顾客，不使用夸张促销或幼儿化语气。
- 称呼：优先使用“你”或省略人称；政策、隐私和订单说明使用准确中性表达。
- 商品信息：先说明商品是什么、适合什么场景，再说明材质、尺寸和注意事项；不写无法证明的功效或材质承诺。
- 订单术语：统一使用“订单请求”“提交请求”“店主确认”，不得使用“已购买”“付款成功”等会造成误解的文案。
- 价格表达：统一为 `CA$12.00` 这类 CAD 格式；不显示人民币或汇率。
- 库存表达：使用“有库存”“仅剩少量”“已售罄”；具体阈值在后续实现阶段确定。
- 图片与无障碍：alt 文案描述商品和视角，不堆叠关键词，不写“图片”。

### 2.1 可供 Phase 2 使用的临时文案

这些是视觉排版用草稿，不代表店主已经批准的正式文案：

- 品牌短句：`把一点可爱和轻松，带进每天的生活。`
- 订单提示：`提交后不会立即付款，也不代表订单已经确认。我们会联系你核对库存、自取或配送安排、税费及最终金额。`
- 自取说明：`可选择本地自取。具体地点和时间将在订单请求确认后提供。`
- 配送说明：`部分温哥华及周边区域可安排本地配送。请在提交请求时填写所在区域和邮编，我们会确认是否可送达及相关费用。`
- 联系时效：`收到请求后，我们会尽快通过你选择的方式联系。`（正式上线前必须替换为店主可承诺的时效）

## 4. 商品与图片资产清单

Phase 0 建议店主选择 5–10 个代表性商品。必须覆盖下列三种规格模式，避免 Phase 3/5 只适配一种商品。

| 商品槽位 | 规格模式 | 选择目的 | 当前状态 |
|---|---|---|---|
| 猫狗茶杯 | 款式 + 颜色，至少 4 个组合 | 验证多规格组合、规格图、独立价格/特价/库存 | 仅有结构名称，无真实资料 |
| 尺寸地毯 | 尺寸，建议至少 3 个值 | 验证尺寸文字、不同规格价格和库存 | 仅有结构名称，无真实资料 |
| 单规格商品 | 无顾客可选项，一个内部默认 variant | 验证不显示空规格选择器 | 未指定真实商品 |
| 代表商品 4 | 店主选择 | 覆盖家居/礼物中的主要分类 | 未提供 |
| 代表商品 5 | 店主选择 | 覆盖不同图片和描述长度 | 未提供 |
| 代表商品 6–10 | 可选 | 提升首页、分类和列表概念的真实性 | 未提供 |

每个入选商品应使用 `docs/PRODUCT_DATA_TEMPLATE.md` 填写，并准备：

- 一张干净、可作为主图的照片；
- 2–5 张细节、侧面、场景或包装图；
- 尺寸参照图（尺寸容易误解时）；
- 不同颜色/款式对应图（外观明显不同时）；
- 每张图的用途、alt 文案和授权来源；
- 原始文件，不要只提供社交平台压缩截图；
- 如需裁剪，保留未裁剪原图。

### 4.1 图片质量不是开发阻塞的边界

- Phase 2 至少需要 Logo 和若干真实商品图，才能形成可信的品牌视觉概念。
- 所有商品都完成专业摄影不是 Phase 1、Phase 2 或后台开发的阻塞条件。
- 正式上线的每个已发布商品至少要有一张清晰、已获授权且与实物一致的主图。
- 供应商图、门店陈列图和手持图可以临时使用，但必须记录授权和后续替换责任。

## 8. Phase 0 验收方法

1. 当前不要求店主复制或人工验证商品模板；模板已经通过三类结构样例完成文档级验证。
2. Phase 2 前提供至少 3 个代表商品的真实照片，并保留商品图片授权来源记录。
3. Phase 8 前确认通知邮箱、联系时效、自取描述和本地配送区域。
4. 后续真实录入商品时，再检查每个可购买组合的唯一 SKU、CAD 价格、库存和启用状态。
