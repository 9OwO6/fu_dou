# Happy Beans 英文网站实施范围（Phase 12）

> 状态：已批准并进入实施
> 决策日期：2026-07-15（America/Vancouver）

## 1. 已确认产品决定

- 公开网站支持 `en` 与 `zh`，默认语言改为英文。
- `/` 重定向至 `/en`；顾客可以从 Header 在 English / 中文之间切换。
- 英文内容缺失时，英文站隐藏对应商品或分类，不以中文静默回退。
- 英文覆盖完整顾客路径：首页、商品列表、分类、集合、商品详情、购物车、订单请求、成功页和顾客确认邮件。
- 后台 URL、导航、按钮、校验和操作说明继续使用中文；店主在中文后台同时录入中英文顾客内容。
- SKU、CAD 价格、原价、库存、销售状态、图片对象、商品 slug 和分类 slug 均为语言无关数据，只维护一份。

## 2. 后台界面结构

### 2.1 商品基础内容

- `网址标识` 作为语言无关字段单独显示，中英文共用。
- 内容字段采用桌面端左右成对、窄屏上下堆叠：
  - 中文标题（必填） / English title（英文展示必填）
  - 中文简短描述 / English short description
  - 中文商品描述 / English description
  - 中文 SEO 标题 / English SEO title
  - 中文 SEO 描述 / English SEO description
- 英文全部留空允许保存草稿；一旦填写任意英文字段，英文标题必须填写。
- 商品列表显示英文完整度，支持店主识别“仅中文”和“中英完整”商品。

### 2.2 分类

- slug、排序和显隐继续共用。
- 中文名称（必填）与 English name（英文展示必填）成对录入。
- 中文描述与 English description 成对录入。
- 英文名称为空的分类不进入 `/en` 分类导航、筛选、页面或 sitemap。

### 2.3 规格、价格和库存

- 每个规格名称增加中文与 English 输入，例如 `颜色 / Color`。
- 每个规格值增加中文与 English 输入，例如 `浅蓝色 / Light Blue`。
- variant 矩阵、SKU、CAD 价格、原价、库存、特价时间和启用状态保持一份。
- 商品只在全部规格名称和值都有英文翻译时进入英文站。

### 2.4 商品图片

- Storage 图片对象、顺序、封面和 variant 关联保持一份。
- 每张图片维护中文 alt 与 English alt。
- 有图片的商品只有在其公开图片均具备 English alt 后才进入英文站。

### 2.5 首页运营

- 模块显隐、排序、选品方式、商品/分类/图片选择和显示数量保持一份。
- 每个模块的标题、说明、按钮、自取/配送说明和 FAQ 同时维护中文与 English。
- 联系邮箱、电话、自取开关和配送开关属于语言无关设置。
- 服务区域说明按语言分别维护。
- 后台继续用中文展示“中文内容”和“English 内容”分区及完整度提示。

## 3. 英文公开条件

### 3.1 商品

英文商品必须同时满足：

1. 商品状态为已发布且发布时间已到。
2. 存在非空 English title。
3. 商品的全部规格名称和规格值都有英文翻译。
4. 已登记公开图片均有 English alt；无图片商品可使用品牌缺图状态。

不满足时，商品仍可正常显示在中文站，但不会进入英文列表、搜索、分类、集合、首页选品或 sitemap；直接访问英文商品 URL 返回英文 404。

### 3.2 分类

分类必须可见且存在非空 English name 才进入英文站。英文分类页只展示同时满足英文商品完整条件的商品。

### 3.3 首页

- migration 提供一套安全的默认英文首页文案，确保 `/en` 首次启用时不会显示中文。
- 店主后续从后台维护正式中英文首页文案。
- 手动选择了英文不完整商品或分类时，英文首页安全跳过该实体，不显示中文替代内容。

## 4. 路由与语言切换

- `/` → `/en`
- `/en/**` 与 `/zh/**` 使用同一 App Router 页面和组件。
- Header 语言切换尽量保留当前 pathname 与 query string。
- 未知 locale 继续返回 404。
- `/admin/**` 不增加 locale 前缀，保持中文后台。

## 5. 订单请求与邮件

- 订单请求保存 `request_locale`（`en | zh`）。
- 提交事务按请求 locale 权威读取商品、规格和值翻译；英文翻译不完整时拒绝提交。
- 订单 item 保存顾客提交时看到的语言快照；SKU、CAD 价格和数量仍来自服务器权威数据。
- 店主通知邮件保持中文后台语境，并标明顾客使用的语言。
- 顾客确认邮件按 `request_locale` 发送中文或英文。
- 两种语言都必须明确“尚未付款、尚未最终确认”。

## 6. SEO 与可访问性

- 每个 locale 使用独立 metadata、canonical 和 `alternates.languages`。
- sitemap 同时列出中文页面和满足英文完整条件的英文实体页面。
- 页面语言属性随 locale 设置；语言切换可键盘操作并有明确可访问名称。
- 英文图片使用 English alt，不使用中文回退。

## 7. 明确不在本 Phase 增加

- 不增加在线支付、顾客账号、多币种、自动税费或运费计算。
- 不翻译后台 UI。
- 不创建第二套商品、variant、库存、价格或图片对象。
- 不引入自动机器翻译或 AI 商品文案。
- 不改变现有自取和小范围本地配送边界。

## 8. 验收门槛

- 数据库从空库 migration 重建成功，RLS、约束、双语保存和英文隐藏规则测试通过。
- `lint`、`typecheck`、单元测试和 production build 通过。
- 真实浏览器验证 `/ → /en`、English/中文切换、英文首页、列表、详情、购物车、订单请求和成功页。
- 验证英文缺失商品不在 `/en` 出现，但继续在 `/zh` 出现。
- 验证英文订单请求保存英文快照，顾客邮件为英文、店主后台仍为中文。
- 桌面、笔记本和手机宽度无页面级横向溢出，浏览器控制台无相关错误。
